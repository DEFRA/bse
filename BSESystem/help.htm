General Information

The page header

A common page header section is displayed at the top of each page in the system.

The centre section of the page header displays the name of the system, i.e. "BSE Database System". If you are using a test or support version of the system, then this information will be displayed immediately below. Finally, the name of the currently active section of the system is displayed.

The right hand block of the page header displays the name of the currently logged-on user and the user group that the user is assigned to. If the user is in one of the VLA user groups and is working on a batch of case details, the current batch number will also be displayed here. Below this are two links that allow you to return to the home page or access this help from anywhere in the system.

Date entry controls

To enter a date in a date entry control, click the calendar button to the right of the text box:

A calendar will be displayed allowing the selection of a date. To close the calendar again without selecting a date, click the calendar button a second time.

Alternatively, a date may be directly typed in the box (in the format dd/mm/yyyy).

Data grids

To select a row in a data grid, click the arrow button at the left-hand end of the row: . The selected row will be highlighted in a different colour.

The buttons at the bottom of the grid may be used to perform row operations:

   Add a new row

   Edit the currently selected row

   Remove the currently selected row from the grid

After pressing the New or Edit button, the grid row will be placed in edit mode, and you can change the values in the cells. After making changes, use these buttons to confirm or cancel those changes:

   Save changes

   Cancel changes - the data in the row will revert to the values that were present before entering edit mode.

The New, Edit or Delete buttons may appear in grey; this indicates that the particular function is unavailable at this time.

If a grid contains more than one page of data, then the navigation buttons at the bottom left of the grid may be used to change between pages:

   Move to the first page

   Move to the previous page

   Move to the next page

   Move to the final page

The page number links, next to the navigation buttons, can be used to move directly to a given numbered page. At the bottom right of the grid the currently displayed page number is shown, along with a count of the total number of pages available.

Some grids will allow sorting by means of clicking on the column headers. If this is the case, then the column headers will appear underlined. The first click on the column header will cause the grid rows to be sorted by that column in ascending order; a second click on the same column will sort in descending order.

Field error messages

If required data is missing, or invalid data has been entered when a form is saved, the error may be indicated by a red star next to the relevant field. Placing the mouse cursor over the star will cause a tooltip message to pop up, indicating the reason for the error: for example, it may indicate that the field is mandatory, or in the case of an invalid entry, it may indicate the required format.

Home Page

The Home link at the top right of every page may be used to return to the Home page at any time.

Batch Number

If you are a VLA user working with a batch of case details, then the batch number box is the place where you specify the batch that you wish to work with. Any cases you enter while the batch number is displayed in the boxes will be added to the batch.

The batch number is in the format yyyy/nnnnnn where yyyy is a four digit year and nnnnnn is an integer up to six digits in length.

If you are beginning work on a new batch then click the Create New button. This will assign a new batch number for you to enter cases against. If the latest batch has no cases recorded against it, then you will be assigned that batch number. The assigned batch number will appear in the text boxes to the left of the button. Note that only the last 3 digits of the batch will be used in the export to OSS function.

To the right hand side of the screen a list of the three most recent batches, together with the number of cases recorded against those batches, is displayed.

RBSE Number

The RBSE number is in the format nn/nn/nnnnn where n is an integer. It is required that you enter the second slash. Any digits you do not enter will be filled with zeros. For example, if you enter 9/87 then it will become 00/09/00087.

For your information, the last used RBSE number and DBSE number for the current and previous year are displayed on the page.

Enter the RBSE number and then click Go to enter details on a case or view details about a case. If you enter an RBSE starting with 63/00 or 23/00 that does not already exist then you will be taken to the Non-GB Case Creation page. If the case already exists then you will be taken to the usual case pages to view the details.

Case Details

The Case pages are navigable using the tabs at the top of the screen, underneath the banner. Every page displays the RBSE of the current case above the tabs.

Every page also displays a batch number button, .  This provides a dropdown table that displays all batch numbers associated with the current case.

Farm

The Farm tab allows entry of details about the farm where the case occurred.

All fields are disabled until you specify a farm by entering a CPHH. You can do this by typing the CPHH into the activated textbox and clicking the Look Up button. If the CPHH already exists in the system then any details held in the database will be entered on the page. If the farm does not exist, then you will be shown a list of farms with the same CPH. You can then select one of these farms. If none of these farms are correct, then you have the option to create a new farm.

When you create a new farm the CPHH will be looked up in the vetnet database. You will be shown a herdmark and numeric herdmark for the farm if present in the database. A message will be shown if the CPHH cannot be found in the vetnet database.

The top section of the page enables you to enter details about the location of the farm. This includes selecting the governing Animal Health Office. The page also enables you to enter a map reference for the farm. If you do not know the map reference then you can click the Estimate button. The estimate button enters a map reference for the centre of the parish specified by the CPHH number. Any map reference you enter is checked to ensure that it lies within the parish boundaries.

The correspondence address enables you to enter separate details for mailings and location. If this is the same as the location address you can click the Use Above Address button to populate the fields.

The Authority County, Local Authority and ADNS Region fields enable you to specify the ADNS Region of the farm - that is, the location that will be reported to Brussels. Picking an Authority County will give you a list of Local Authorities in that county. Picking a Local Authority will give you a list of ADNS Regions in that Authority. You must select an ADNS Region for the farm if it is within Great Britain; otherwise, any cases on the farm could not be reported to Brussels.

The Herdmark and Numeric Herdmark fields enable you to enter details about the herdmark for the farm. If you have entered a new farm and the details have been found in the vetnet database then these fields might already be populated. The herdmark must consist of up to four letters followed by up to four numbers. The numeric herdmark is in the format nnnnnn, six integers.

The section above the tables enables you to enter details about the type of herd on the farm and whether or not the farm is a dealer.

The Related Farms table enables you to enter the CPHH of farms that have a connection to the case farm. If a case has been reported at the CPHH entered then the farm details held in the system will be displayed. If no details for the farm are held in the system then the text 'BSE Free' will be displayed next to the CPHH.

The Herd Size table holds data about the size of the herd in preceding years. You are required to enter a Year and Herd Size. If you enter a value in the Lactation fields then all empty fields will be filled with zeros. If the values in the Lactation fields do not add to the value entered in the Herd Size field then a warning will be displayed.

The button at the bottom left of the Farm tab takes you to the Farm Audit log. This enables you to see changes made to the Farm details, who made the changes and when the changes were made.

Case (DEFRA)

The Case (DEFRA) tab holds data that DEFRA have entered on a case.

The top section enables you to enter the eartag of the animal and the dates the Form A, Form B, and Form C were received.

Eartag : The eartag should be entered with a suitable delimiter between the herdmark and the individual animal Id, unless it is an all numeric eartag, in which case it should be prefixed with the country of origin. Eg AB123-234 or UK123456789012. An eartag validation routine (provided by ITD) is used to determine the format of the eartag (Pre-Barimo, Barimo, Numeric etc). Identifiable non-numeric formats are split into herdmark and individual Id and are held separately on the database for the purposes of searching. If the format cannot be determined it is assumed to be “Free-format” and the whole eartag is stored in the individual Id field. Numeric eartags are also held in the individual Id field.

Important: Some old eartag formats are not handled well by the validation routine and need special data entry rules. Where the first two letters of a pre-barimo format British eartag could indicate a country code, the validation routine will fail and the eartag will not be stored correctly. These eartags should be prefixed with a dash or an asterisk to indicate that the country code is missing.

e.g. “DE494-394” has to be entered as “–DE494-394”

Letters which have to be prefixed are:

AT, BE, DE, DK, EL, ES, FR, IE, LU, SE, NL, PT AND SE

Updates have been made to the ear tag validation to allow entry of the ISO electronic ear tag format and to cater for existing validation detailed (above).

New formats are

Numeric ISO format

[3 numerical digits][0 or 1 or 2 or _ or space] [Herd Mark (Hn6)] [Animal Identifier (An5)]

Example: GB0 703412 70001

Alpha-numeric ISO format

[2 alphabetical characters][0 or 1 or 2 or _ or space] [Herd Mark (Hn6)] [Animal Identifier (An5)]

Example: 8260 703412 70001

The next section details the tests conducted on the animal and the results that were obtained. If a Final Result has been entered against the case then this will be displayed here.

You can then enter details about the report location for the case and the ageof the animal (if known). The checkboxes at the bottom of the screen enable you to enter details about what paperwork has been received.

The button at the bottom left of the Case (DEFRA) tab takes you to the Case Audit log. This enables you to see what changes have been made to the Case details, by whom and when.

Case (VLA)

The Case (VLA) tab is designed for data specific to the VLA.

The top section enables you to enter details about the animal. This data is the same as on the DEFRA and BAB tabs so might be populated when you first visit the page.

The purchased section only becomes enabled if the origin of the animal is Purchased.  If the origin is changed away from purchased and data has been entered then it will be erased. Changing the origin away from purchased might also have an effect on the Traced data entered on the BAB tab. You will be warned if any data will be erased by your action.

If you enter a Date of Birth and a Date Purchased then the Age Purchased will be calculated. Age purchased will always be displayed in months. However, you can enter the age in years and months. Clicking the Calculate button will convert the figure into months.

The next section enables you to enter details about the onset of BSE. If you have entered an Onset Date and a Date of Birth then the Onset Age will be calculated. You can also enter the Onset Age in years and months. Clicking Calculate will display the Onset Age in months. The value for Months Post Pregnant must be between one and three. The Months Pregnant must be a value between one and nine.

The table at the bottom of the page enables you to enter details about Other Owners.

"Previous" indicates the previous owner of the animal and should be entered for purchased cases.

"Post" indicates the owner of the animal at the time of notification if the case has been moved to the previous owner (usually because the current owner was a dealer).

"56 Day rule" indicates the owner of the animal at the time of notification if the case has been moved to the previous owner as a result of the 56 day ownership rule.

The button at the bottom left of the Case (VLA) tab takes you to the Case Audit log.  This enables you to see what changes have been made to the Case details, by whom and when.

Relations

The Relations tab enables you to enter information about animals related to the case. The page is divided into four sections: Case Herdbook, Dam Details, Sire Details, and Other Relations.

Case Herdbook

The Case Herdbook section enables you to view and edit the case's herdbook number.

Sire Details

The Sire Details section of the page enables you to enter information about the case's sire. If the case you are working with does not have a sire recorded, then you can enter information in one or more of the following fields: Eartag, Name, RBSE, Herdbook. Then click the 'Look Up' button to search the database to try and find a male animal that matches the criteria you entered. If you provided an RBSE, only the RBSE information will be taken into account in the search.

If a single animal matching the details you entered is found, then he will be recorded as the sire for the case and his details displayed. Otherwise, you will be taken to a page listing possible matches for the sire. You can select an animal from the table and click 'Use Selected'; then this animal will be recorded as the sire for the case and his details displayed. If none of the suggested animals is the correct one, you can click the 'New' button to create a new sire with the eartag, name and herdbook you entered.

When a sire is recorded against the case, his details are displayed, including a count of the number of offspring he has, and his fate. If the sire is not a case, his eartag, name, herdbook and date of birth can be edited here. If he is a case, only the name can be edited - the other details must be amended by using the Case Entry screens with his RBSE. Note that if any of these details are changed, the changes will affect every case whose sire this animal is - the offspring count will show you how many other cases may be affected by the changes. Approximate birth dates can be provided by entering only a year, or a month and year, rather than a full day/month/year combination.

If the sire is himself a case, then the 'View Sire's Relations' buttons will be enabled. Clicking this will bring up a popup window displaying information about the relations that are recorded against the sire's case. This will enable you to check that when a sire/offspring pair are both cases, the relation is recorded correctly against each of them.

Dam Details

The Dam Details section enables you to enter information about the case's dam. If the case you are working with does not have a dam recorded, then you can enter information in one or more of the following fields: Eartag, Name, RBSE, Herdbook. Then click the 'Look Up' button to search the database to try and find a female animal that matches the criteria you entered. If you provided an RBSE, only the RBSE information will be taken into account in the search.

If a single animal matching the details you entered is found, then she will be recorded as the dam for the case and her details displayed. Otherwise, you will be taken to a page listing possible matches for the dam. You can select an animal from the table and click 'Use Selected'; then this animal will be recorded as the dam for the case and her details displayed. If none of the suggested animals is the correct one, you can click the 'New' button to create a new dam with the eartag, name and herdbook you entered.

When a dam is recorded against the case, her details are displayed, including a count of the number of offspring she has, her fate and final result. If the dam is not a case, her eartag, name, herdbook and date of birth can be edited here. If she is a case, only her name can be edited - the other details must be amended by using the Case Entry screens with her RBSE. Note that if any of these details are changed, the changes will affect every case whose dam this animal is - the offspring count will show you how many other cases may be affected by the changes. Approximate birth dates can be provided by entering only a year, or a month and year, rather than a full day/month/year combination.

If the dam is herself a case, then the 'View Dam's Relations' buttons will be enabled. Clicking this will bring up a popup window displaying information about the relations that are recorded against the dam's case. This will enable you to check that when a dam/offspring pair are both cases, the relation is recorded correctly against each of them.

The final field in the Dam Details section is the Dam Status dropdown list. This enables you to indicate the status of the dam at the time the case was reported. This information is recorded against the case, not the dam, because it refers to the status of the dam at the time the case was reported. It is possible to indicate a dam status even when the identity of the dam is unknown.

Other Relations

The fourth section enables you to enter the details of the case's other relations - chiefly offspring, but also twins and full sisters.

To add a relation, enter the details in the fields at the bottom of the page and then click 'Add New' to add it to the table. To edit a row in the table, select the row as normal. The details will appear in the fields at the bottom of the page. You can then make changes and select to update the row or delete the row.

You must select a relation type for each relation that you enter. If the relation is also a case, enter the RBSE number of the case. The other details of the animal will be filled in for you from the case details associated with the animal you specified, and cannot be edited. Otherwise, for related animals that are not cases, the sex, fate, eartag, birth date, left date and sire name can be entered.

If a relation is a case, the 'View Relations' button will be enabled. Clicking this will bring up a popup window displaying information about the relations that are recorded against the relation's case. This will enable you to check that both sides of a relationship involving two cases are recorded correctly against each of them.

Export To ADNS

The ADNS Export pages enable newly confirmed cases of BSE to be reported via email to the Animal Disease Notification System (ADNS) in Brussels.

There are differing processes for GB cases, Channel Island and Isle of Man cases, and Northern Ireland cases. The ADNS menu allows you to choose one of these three export options.

ADNS Export - GB Cases

To initiate an export of GB case data, you must provide an email reference (eg. DBSE00827Sep3) and a starting ADNS reference number. The ADNS reference number consists of a four digit year and a number with up to five digits. Once these two items of data have been provided, you can click the 'Generate Report' button to see a summary of the information that will be exported to Brussels.

A list of missing cases displays the RBSE numbers of positive cases that cannot be placed into an ADNS region by the database. If any missing cases appear, these should be investigated to determine their ADNS location. Below the missing cases list is a summary indicating the start and end ADNS reference numbers that will be allocated, and a table showing how many cases will be reported in each of the ADNS regions.

Below this summary is a form showing the contents of the email that will be sent - the From address, the To address, the Subject and the Body. If this data is all as expected, you can click the 'Send Email' button. This will send an email to the Brussels ADNS system and also send a copy to your email address.

ADNS Export - Channel Island Cases

As with the GB export, you must provide an email reference and a starting ADNS reference number for the export. However, rather than the database determining the cases to be reported, you must specify the number of cases to report in Jersey, Guernsey and the Isle of Man by entering figures into the three textboxes provided for this purpose.

Once this is done, you can click on the 'Generate Report' button to display a summary indicating the start and end ADNS reference numbers that will be allocated, and a table showing how many cases will be reported in each of the ADNS regions.

Below this summary is a form showing the contents of the email that will be sent - the From address, the To address, the Subject and the Body. If this data is all as expected, you can click the 'Send Email' button. This will send an email to the Brussels ADNS system and also send a copy to your email address.

ADNS Export - Northern Ireland Cases

Enter an email reference for the export. Then, for each case to report, enter the ADNS reference number, region, and confirmation date and click the 'Add To Grid' button. The case data will be added to the grid. If you wish to remove data that you have added to the grid, delete the row in the normal way.

Then, when you have added all the cases that you need to export to the grid, click the 'Generate Report' button to display a summary indicating the start and end ADNS reference numbers that will be allocated, and a table showing how many cases will be reported in each of the ADNS regions.

Below this summary is a form showing the contents of the email that will be sent - the From address, the To address, the Subject and the Body. If this data is all as expected, you can click the 'Send Email' button. This will send an email to the Brussels ADNS system and also send a copy to your email address.

Final Result Entry

The Final Result Entry page enables you to record a Final Result against a case. To enable the controls you need to enter the RBSE of a case already in the system. The details for this case will then be displayed on the screen.

If a verdict of positive has been recorded against the case then the Final Result Date and DBSE will be displayed. The Download button will also become enabled and allow you to save the result memo. If the Result is negative then the Download button will be enabled and a Final Result Date will be shown but no DBSE will be displayed. If you do not need to save the memo and will simply print then you can use the Print button. This will display a page in your browser which you can print.

Once you have selected a Final Result it will not be recorded against the animal until you click Save. If you change the result on a case that already has a Final Result then you will need to save the changes before you can download a new copy of the Result Memo.

Once you have saved a Final Result against a case you will be taken to a confirmation page. This will display the RBSE of the case you have just entered data against and the DBSE if relevant. The Download button enables you to download the Result Memo for the case whilst clicking Print will enable you to print the memo from your browser. The Add another result button takes you back to the Final Result Entry page so you can add more results. This page also has a link back to the Home page.

Export To OSS

The Export To OSS pages enable the generation of files for import into the Optical Storage System.

BSE1

The BSE1 page enables the generation of a file for importing into the Optical Storage System. You must enter a Batch Number to generate the file. If the Batch Number exists in the system a separate window will open and you will be asked if you want to Open or Save the file. Click Save, navigate to the directory you want the file to be located in and change the document type from Text Document to All Files.

The OSS system only takes files with 3 digit endings. Therefore any batch numbers where the number is greater than three characters will be truncated. For instance 2001/1987 gets truncated to 2001/987. If this occurs then a message will be displayed on the BSE1 screen. Once you have saved the file you can close the new window.

BSE1b

The BSE1b page enables you to add cases to a BSE1b Batch.

You must enter the RBSE of a Case in the system. A new BSE1b Batch Number will then be created and displayed on the page. Any further RBSE numbers you enter will be recorded against the Batch Number.

You can remove Rows by selecting and deleting them in the Grid. This is done in the usual way.

Once all the Cases have been added to the table click Export to OSS to record the Cases against the Batch Number. At the same time a separate window will open and you will be asked if you want to Open or Save the export file. Click Save, navigate to the directory you want the file to be located in and change the document type from Text Document to All Files.

The Maintenance pages are available to enable Maintenance users to make changes to farm and case records.

Move Case

The Move Case enables you to transfer a case from one farm to another.

You must enter the details of the case and then click Look Up. Details of the case will be displayed and the New CPHH textbox will become enabled. You can then enter the New CPHH and click Check. You will then be shown details of the farm you have selected and can click Ok. The RBSE and CPHH you enter must already exist on the system.

If the old CPHH associated with the case is not related to another case then the farm will be deleted from the database. A message will be displayed in this instance.

No changes will be made to the database until you click Ok. You will be taken to a confirmation page once the change has taken place.

Delete Case

The Delete Case page enables you to delete a case.

You must enter the RBSE of a case already in the system. If this is the only case on a Farm then details of the Farm will be deleted too. A message will be displayed if the Farm is to be deleted.

No changes will be made to the database until you click Ok. You will be taken to a confirmation page once the change has taken place.

User Maintenance

This page enables you to add and remove users from the system. VLA Maintenance users are the only users with access to this page.

You cannot delete a user from the database as this will affect the Audit Log. However, you can make the user inactive. The e-mail address is used for the ADNS export.

Application Error

This screen is displayed if the system has encountered an unexpected problem. Details of the error are shown, which a system administrator may be able to use to determine the cause of the problem.

The information displayed will also have been recorded in the system log, which the system administrator can view through the View Log screen. To help the administrator determine the cause of the problem, you can record additional information about what you were doing when the error occurred in the text box at the bottom of the page. Click the Save button and the information you entered will also be saved to the system log, for later analysis by the system administrator.

After an error has occurred, it is advisable to use the Home link in the page header to return to the Home screen.
